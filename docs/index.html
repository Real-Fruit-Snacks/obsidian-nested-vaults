Capabilities

Everything scopes. Nothing leaks.

Every part of the Obsidian UI is made aware of your boundary. This is not just a file filter -- it is a complete workspace isolation layer.

File Explorer scoping

Folders and files outside your sub-vault vanish from the sidebar instantly. Only the content you chose to focus on remains visible.

Search and Graph injection

The plugin automatically enforces a path: filter into Core Search and Graph View. It resists overrides and re-applies on every mutation.

Tags and Backlinks filtering

Tags and backlinks belonging to notes outside your scope are hidden. You only see the context relevant to your focused project.

Navigation blocking

Click a link that leads outside the boundary? The plugin intercepts it, blocks the navigation, and closes the tab before you leave your focus zone.

Automatic note capture

New notes created while a sub-vault is active are automatically moved into the scoped folder. Nothing escapes the boundary.

Global allowed folders

Whitelist folders like Attachments or Templates so shared resources remain accessible no matter which sub-vault you are inside.

Workflow

Three clicks. Total isolation.

1

Right-click any folder

Select Set as active sub-vault from the context menu, or use the Command Palette to pick a folder with fuzzy search.

2

Your workspace contracts

The File Explorer, Search, Tags, Graph View, and Backlinks all instantly scope down. Everything outside the boundary disappears.

3

Leave when you are done

Click the ribbon icon, click the status bar indicator, or run the Clear active sub-vault command. Everything returns to normal.

Get Started

Install in under a minute.

Recommended

BRAT

  1. Install the BRAT community plugin.
  2. Open BRAT settings and click Add Beta plugin.
  3. Paste this repository URL.
  4. Enable Nested Vaults in your plugin list.

Manual

  1. Download main.js, manifest.json, and styles.css from the latest release.
  2. Create .obsidian/plugins/obsidian-nested-vaults/ inside your vault.
  3. Place the three files inside that folder.
  4. Reload Obsidian and enable the plugin.
